2. Полная структура папки militarydepot (7 файлов)
HarmonyPatches.cs — Общие патчи для работы системы склада (перехват функций инвентаря/обмена).

MilitaryDepotActions.cs — Технические действия: открытие меню (K), быстрая сортировка (O).

MilitaryDepotBehavior.cs — Привязка логики склада к жизненному циклу кампании.

MilitaryDepotCache.cs — Кэширование данных о снаряжении юнитов для оптимизации производительности.

MilitaryDepotLogic.cs — "Сердце" склада: расчет планов экипировки и распределение предметов.

MilitaryDepotLogs.cs — Система отладки и вывода информации в консоль/чат именно по складу.

MissionPatches.cs — Патчи для миссий (сцен сражений или городов), если склад должен влиять на снаряжение во время боя.

3. Структура папки militarytrade (2 файла)
militarydepottrade.cs — Логика «Резерв + 15%» и умная сортировка для продажи ИИ.

tradepatches.cs — Harmony-патч на OnSettlementEntered для автоматизации торговли лордов.

5. В разработке (In Progress)
Логика активных закупок ИИ (AI Smart Buy):

[ ] Анализ дефицита: Интеграция с MilitaryDepotLogic для выявления "дыр" в экипировке отряда (например: "в отряде 20 стрелков, а луков в инвентаре только 5").

[ ] Поиск на рынке: Обучение лорда проверять ассортимент города при входе в поселение на предмет наличия нужных типов снаряжения.

[ ] Бюджетирование: Установка лимита трат.

Личный лимит лорда («Операционный капитал»)
У каждого лорда должен быть «здоровый» предел наличности.  10,000 динар.
		
Проверка бедности (Refill): При входе в город, если Hero.Gold < 2000 и Clan.Gold > 50000 — перевести от клана лорду 5000.

Бюджет на закупку (Shopping): Лорд выделяет на покупку снаряжения фиксированный процент (например, 30%) от того, что у него есть сверх лимита выживания.

Передача излишков (Tribute to Clan): Если после всех покупок у лорда всё еще аномально много денег (больше лимита в 15,000), остаток перечисляется в Clan.Gold.



[ ] Приоритет качества: Логика покупки лучшего из доступного (по цене/характеристикам) в рамках бюджета для закрытия самых критических нужд отряда.

[ ] Интеграция алгоритма «Щит-Оружие-Броня»:

[ ] Создание логики «Сначала щит» для всех нуждающихся юнитов.

[ ] Метод CalculateEfficiencyScore: оценка предмета через соотношение статов к цене.

[ ] Метод HandleItemReplacement: автоматическая продажа старого снаряжения при замене на лучшее.

[ ] Проверка требований: навыки (Bow/Riding) и специфика (Mounted/Thrust).
